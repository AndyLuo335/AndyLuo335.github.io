Arrays

While doing a few of the leetcode challenges listed for static arrays, I realized an insight for dealing with an array where you want to remove certain values and return a smaller array. We use two indexer's i and j, i is used to iterate through the list while j is used to define the end of your "good zone".
Whenever i encounters something that belongs in the good zone, increment j by one and add it to the end of the good zone.
In hindsight this is only useful if you need to modify and return the input array with an int :P

  • https://leetcode.com/problems/remove-duplicates-from-sorted-array/
  • https://leetcode.com/problems/remove-element/

When trying to keep track of the minimum value in a stack, instead of keeping track of the min overall, we should keep track of it in frames or snapshots.
Every time we push something into the stack, we pair that value with the current minimum of the stack. So, if we pop the top element, the next top element has the prerecorded current min already!

  • https://leetcode.com/problems/min-stack/
